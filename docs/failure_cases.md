# Failure Cases

> 记录 schema / prompt / pipeline 无法处理的情况。
> 这些数据对理解 framework 的边界至关重要。

## How to record

每遇到一个失败案例，按以下格式追加：

```markdown
### YYYY-MM-DD: [Brief title]

**Source:** [book / chapter / section]
**Problem:** [What could not be expressed or processed]
**Root cause:** [Schema gap / Prompt ambiguity / Table format / Figure / Other]
**Resolution:** [How it was fixed, or "Not yet resolved"]
**Impact:** [What would have been lost without this fix]
```
